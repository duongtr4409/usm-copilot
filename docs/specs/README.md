# Feature Specifications
**Owner**: @BA
**Consumer**: @Tech-lead, @QA-Tester

This directory holds all feature specification documents produced by `@BA`.

## Naming Convention
```
TASK-{number}-{feature-slug}.md
```

Examples:
- `TASK-001-user-authentication.md`
- `TASK-002-product-catalog.md`
- `TASK-003-order-management.md`

## Template

Use the template in [BA.agent.md](../../.github/copilot-agents/BA.agent.md).

All spec files must include:
- Feature overview
- User personas
- User Stories (US-XXX) with Gherkin Acceptance Criteria
- Data model (conceptual)
- Non-functional requirements
- Open questions log

## Status Lifecycle
`DRAFT` → *(PMO approval)* → `APPROVED` → *(Development starts)* → `IMPLEMENTED`
